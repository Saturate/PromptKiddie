# Enigma Retro - 2026-07-13

HTB Easy Linux machine. NFS leak -> email creds -> OpenSTAManager RCE -> DB hash crack ->
OliveTin command injection to root. Both flags captured in ~90 minutes wall time with 3
agents and ~500 tool calls.

## What happened

First engagement using the new reactive playbook supervisor. The supervisor auto-dispatched
rustscan, nmap, whatweb, ffuf, and nuclei within 90 seconds, discovering 13 open ports and
the hostname `enigma.htb`. The enum agent found NFS creds, chained through IMAP to discover
a hidden support portal (OpenSTAManager), and recovered admin credentials from an internal
email. Two exploit agents (one burned its budget, the second finished the job) completed
the full chain from webshell to root.

## Attack chain (what worked)

1. **NFS share leak.** `/srv/nfs/onboarding` exported world-readable. Contained
   `New_Employee_Access.pdf` with webmail creds `kevin:Enigma2024!`.

2. **IMAP email chain.** Logged in as kevin, then sarah (same default password). Sarah's
   inbox contained an email from `it@enigma.htb` with admin creds for the support portal:
   `admin:Ne3s4rtars78s` at `support_001.enigma.htb`.

3. **OpenSTAManager ZIP upload RCE.** The Updates module (id_module=6) accepts ZIP uploads
   that extract to the web root. Uploaded a PHP webshell, achieved RCE as www-data.
   (Note: the intended exploit was CVE-2025-69212 via .p7m filename injection, but the
   generic ZIP upload also worked.)

4. **DB credential harvesting.** Read `/var/www/html/openstamanager/config.inc.php` for
   MySQL creds `brollin:Fri3nds@9099`. Queried `zz_users` table, found haris's bcrypt hash.

5. **Hash crack + user flag.** John the Ripper cracked the bcrypt hash in 8 seconds:
   `haris:bestfriends`. Used `su -s /bin/bash haris` from the webshell to read user.txt.

6. **SSH key plant.** Generated ed25519 key pair, deployed via webshell+su for stable access.

7. **OliveTin privesc (CVE-2026-27626).** OliveTin v3000.10.0 running as root on
   localhost:1337, no auth required. The `backup_database` action takes a `db_pass` argument
   of type `password` that is not sanitized. Injected `x'; cat /root/root.txt #` via the
   ConnectRPC API (`/api/olivetin.api.v1.OliveTinApiService/StartActionAndWait`). Root flag
   returned in the action output.

## What worked well

- **Supervisor auto-dispatch.** 13 ports discovered, 7 tools dispatched, all within 90
  seconds. Zero orchestrator intervention for recon.
- **Enum agent efficiency.** Found NFS creds, email chain, all 3 vhosts, and the admin
  password within 137 tool calls. Complete attack surface mapped.
- **Delegation hook fired.** At 20 inline orchestrator commands, the hook blocked further
  tool use and forced agent spawning. Worked exactly as designed.
- **Second exploit agent completed the chain.** Given a tighter brief (80 calls) and the
  context from agent 1, it went from zero to root in 117 tool calls.
- **Activity logging.** ~400 commands captured across all agents via pk exec.

## What broke (tooling gaps)

### 1. First exploit agent wasted its budget on broad enumeration

The agent had 200 tool calls and spent most of them listing systemd services, reading cron
configs, and enumerating file permissions instead of trying `cat /home/*/user.txt` first.
On an easy box, obvious paths should be tried within the first 5 commands.

**Fix:** Add "try obvious flag paths first" to exploit agent brief for CTF engagements.
Reduce default budget to 80 for easy boxes.

### 2. Orchestrator misidentified the target

The webshell returned a Docker container hostname (`cf6b9cda951f`) and Kali OS. The
orchestrator concluded this was the attackbox and declared the RCE a false positive. In
reality, the target runs OpenSTAManager in a Docker container.

**Fix:** Target verification should compare the webshell's network-facing IP (curl
ifconfig.me or check which IP the HTTP response came from) against the target IP, not
just hostname/OS. Docker container hostnames are random hex and don't prove identity.

### 3. /etc/hosts appended 20+ times

Multiple agents each ran `echo "... enigma.htb" >> /etc/hosts` without checking if the
entry already existed. The hosts file grew to ~5KB of duplicates.

**Fix:** Always use `grep -q hostname /etc/hosts || echo "..." >> /etc/hosts`.

### 4. Spawn retry spam (fixed mid-engagement)

The supervisor tried to `pk spawn` agent containers 13 times with an image that didn't
exist (`pk-agent-attack`). Fixed by adding a 2-retry cap per action+image pair. The
fallback correctly sent tasks to the inbox.

### 5. Agent didn't use pk webshell register

Despite the brief telling it to register the webshell with `pk webshell register`, the
agent used raw curl for all target interaction. The webshell commands were logged via
pk exec (the curl wrapper), but the structured webshell tracking was bypassed.

**Fix:** Stronger language in the brief, or auto-detect webshell URLs in pk exec output.
